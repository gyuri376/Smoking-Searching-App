# 🚬 여기흡연 (Smoking Searching App)

**주변의 흡연 구역을 쉽고 빠르게 찾는 지도 기반 앱**

여기흡연은 위치 기반 서비스를 활용하여 사용자 주변의 흡연 구역을 실시간으로 검색할 수 있는 모바일 웹 애플리케이션입니다. Google Maps를 기반으로 흡연부스의 위치, 운영 상태, 혼잡도 정보를 한눈에 확인하고, 사용자가 새로운 장소를 제보하며 커뮤니티를 구축할 수 있습니다.

---

## 🎯 핵심 기능

### 1. **홈 - 지도 기반 주변 흡연구역 검색** (`/`)

- **현재 위치 기반 검색**: "위치 허용" 버튼 클릭 후 자동으로 주변 흡연 구역 로드
- **인터랙티브 지도**: Google Maps에서 흡연부스 위치를 핀으로 표시
- **카드 리스트**: 지도 아래 흡연 구역 정보(거리, 운영 상태, 혼잡도)를 스크롤로 조회
- **상세 정보 팝업**: 특정 흡연 구역 클릭 시 이름, 정보, 시설, 거리 표시

**작동 원리**:
1. 사용자 위치 권한 요청 (Geolocation API)
2. 백엔드에서 반경 1km 내 흡연구역 데이터 조회
3. Google Maps에 마커로 시각화
4. 사용자 선택 시 상세 정보 표시

---

### 2. **리스트 - 흡연구역 목록 정렬 및 필터** (`/list`)

- **거리순 정렬** (기본): 가장 가까운 흡연 구역부터 표시
- **평점순 정렬**: 사용자 평가가 높은 순서대로 정렬
- **혼잡도순 정렬**: 혼잡도가 낮은 구역부터 표시
- **확장 가능**: 스마트 필터(시설, 운영 시간 등) 추가 예정

**작동 원리**:
- 사용자가 정렬 옵션 선택 시 클라이언트에서 즉시 재정렬
- 거리/혼잡도는 수치화하여 알고리즘 기반 정렬 수행

---

### 3. **즐겨찾기 - 자주 방문하는 흡연구역 저장** (`/favorites`)

- **로그인 필수**: 카카오 인증 후 이용 가능
- **즐겨찾기 추가/삭제**: 홈 또는 리스트에서 ⭐ 버튼으로 저장
- **알림 히스토리**: 즐겨찾기한 구역의 운영 상태 변화 알림 (UI 준비 완료)
- **순서 편집**: 즐겨찾기 목록의 순서 커스터마이징 (UI 준비 완료)

**데이터 저장**:
- AppContext를 통해 로컬 상태 관리
- 향후 백엔드 DB 연동 예정

---

### 4. **최근 기록 - 조회 히스토리** (`/recent`)

- **자동 기록**: 사용자가 클릭한 흡연 구역 자동 저장
- **빠른 재접근**: 최근에 본 장소를 빠르게 다시 확인 가능
- **로컬스토리지 저장**: 앱 재방문 시에도 기록 유지

---

### 5. **커뮤니티 - 새로운 장소 제보** (`/community`)

- **신규 장소 제보**: 사용자가 발견한 새로운 흡연 구역 정보 입력
  - 📍 장소명
  - ⭐ 평점 (별 5개)
  - 🏠 시설 정보 (지붕, 의자, 재떨이, 환기, 콘센트 등)
  - 📸 사진 업로드
- **리뷰 작성**: 기존 흡연 구역에 대한 리뷰 및 최신 정보 업데이트
- **커뮤니티 기반**: 모든 사용자가 정보를 공유하여 정확성 향상

**작동 원리**:
1. 폼 데이터 수집
2. 이미지 파일 처리 (FormData)
3. 백엔드 API로 전송
4. 게시판/알림으로 피드백 반영

---

### 6. **마이페이지 - 사용자 인증** (`/mypage`)

- **카카오 소셜 로그인**: 카카오 인증으로 간편 로그인
- **사용자 프로필**: 로그인 후 닉네임 표시
- **로그인 상태 관리**: JWT 토큰 저장 및 검증
- **권한 기반 기능**: 로그인 사용자만 즐겨찾기, 리뷰 작성 가능

**인증 플로우**:
1. 카카오 인가 코드 요청
2. 프론트엔드에서 코드 수신
3. 백엔드에 JWT 토큰 발급 요청
4. AppContext에 토큰 저장
5. 이후 API 호출 시 Authorization 헤더 포함

## 🚀 로컬에서 실행하기

### 필수 요구사항
- Node.js 16.x 이상
- npm 또는 yarn

### 설치 및 실행

```bash
# 1. 저장소 클론
git clone <repository-url>
cd Smoking-Searching-App

# 2. 의존성 설치
npm install

# 3. 환경 변수 설정
# .env.local 파일에 다음 항목 추가:
# - NEXT_PUBLIC_GOOGLE_MAPS_API_KEY=your_google_maps_api_key
# - NEXT_PUBLIC_KAKAO_CLIENT_ID=your_kakao_client_id
# - NEXT_PUBLIC_BACKEND_URL=http://localhost:4000

# 4. 개발 서버 시작
npm run dev
# 브라우저에서 http://localhost:3000 접속

# 5. 프로덕션 빌드
npm run build
npm run start
```

## 🔄 작동 원리 및 데이터 흐름

```
사용자 상호작용
    ↓
React 컴포넌트 (상태 변경)
    ↓
Context API / useState (상태 관리)
    ↓
API 호출 (src/api.js)
    ↓
Mock Data / 백엔드 API
    ↓
데이터 렌더링 (UI 업데이트)
```

### 예시: 홈 페이지 흐름

1. **사용자 "위치 허용" 클릭**
   ```javascript
   navigator.geolocation.getCurrentPosition(callback)
   ```

2. **위치 좌표 받기**
   ```javascript
   setPosition({ lat: 35.88, lng: 128.59 })
   ```

3. **주변 흡연구역 API 호출**
   ```javascript
   fetchNearbySmokingAreas(lat, lng) → API 응답
   ```

4. **데이터 상태 업데이트**
   ```javascript
   setSpots(data) → 리렌더링
   ```

5. **Google Maps 마커 표시**
   ```javascript
   <GoogleMapView center={position} spots={spots} />
   ```
